⌨️ COMANDOS RÁPIDOS
================================================================================

BACKUP:
  mysqldump -u root -p hirata_cars > backup_$(date +%Y%m%d_%H%M%S).sql

PRÉ-VALIDAÇÃO:
  php scripts/pre-validate-standardization.php

EXECUTAR SCRIPT:
  mysql -u root -p hirata_cars < scripts/standardize-camelcase.sql

PÓS-VALIDAÇÃO:
  php scripts/post-validate-standardization.php

ROLLBACK:
  mysql -u root -p hirata_cars < backup_20260421_143045.sql

CONTAR UNDERSCORES RESTANTES:
  mysql -u root -p hirata_cars -e "SELECT COUNT(*) FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_SCHEMA = 'hirata_cars' AND COLUMN_NAME LIKE '%\_%';"

LISTAR COLUNAS COM UNDERSCORES:
  mysql -u root -p hirata_cars -e "SELECT TABLE_NAME, COLUMN_NAME FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_SCHEMA = 'hirata_cars' AND COLUMN_NAME LIKE '%\_%' ORDER BY TABLE_NAME;"

🚨 PROBLEMAS COMUNS & SOLUÇÕES
================================================================================

PROBLEMA                            SOLUÇÃO
──────────────────────────────────────────────────────────────────────────
MySQL não conecta                   systemctl start mysql
                                    mysql -u root -p -e "SELECT 1;"

ERROR 1064 no script SQL            Parar script, restaurar backup
                                    Verificar linha exata do erro

ERROR 1452 Foreign Key              Já tratado (SET FK_CHECKS = 0)
                                    Se persistir: rollback completo

Aplicação não conecta pós-          Verificar entities.js atualizado
padronização                        Verificar .env credenciais
                                    Reiniciar backend

Underscores ainda existem           Rodar post-validate script
                                    Executar ALTERs manuais
                                    Contactar DBA

Índices com nomes antigos           Normalizar via RENAME INDEX
                                    Ou ignorar se funcionam
